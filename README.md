**Лаборатороная работа №2: Договоры банка. Услуги - набор услуг банка, заявки - заявка на подключение к обслуживанию.** 

## Цель работы
Разработка структуры базы данных и ее подключение к бэкенду

## Описание
В данной лабораторной работе реализована полноценная база данных PostgreSQL, интегрированная с Django через ORM. 
Данные услуг и заявок хранятся в таблицах, а не в коллекциях. 
Добавлены новые функции:
- Поиск услуг через ORM (фильтрация по названию).
- Добавление услуг в заявку (корзину) через ORM.
- Логическое удаление заявки через сырой SQL UPDATE.
- Просмотр заявки с таблицей услуг (изображения, балансовые и банковские счета).
- Автоматическое создание черновика заявки при добавлении первой услуги.
- Удалённые заявки недоступны для просмотра.
**Поля по предметной области** 
услуги: ставка, сумма, срок, балансовый счёт, описание, изображение, видео.

Всего реализовано **5 HTTP методов**: 
три GET (список услуг, детальная услуга, просмотр заявки) 
два POST (добавление в заявку через ORM, удаление заявки через SQL UPDATE)

**Три шаблона** (страницы) полностью работают с БД через Django ORM:
- bankservice_list.html – список услуг с поиском.
- bankservice_detail.html – детальная карточка услуги.
- bank_request.html – просмотр текущей заявки (корзины).

## Структура базы данных (ER-диаграмма)
- Таблица `auth_user` (системная Django)
- Таблица `bmstu_lab_bankservice` (услуги)
- Таблица `bmstu_lab_bankrequest` (заявки, 5 статусов)
- Таблица `bmstu_lab_bankserviceinrequest` (связь многие-ко-многим с доп. полями: количество, комментарий, банковский счёт)

**Каскадное удаление запрещено** (`DO_NOTHING`).

## Карточка заявки (корзина) не активна, если нет черновика	
В bankservice_list условие {% if bank_request %} – отображает корзину и кнопку удаления только когда есть активный черновик.

## 5 статусов заявки: 
DRAFT, DELETED, FORMED, COMPLETED, REJECTED (в модели BankRequest.Status)

## переход по URL удалённой заявки
ввод url удаленной заявки переводит на начальную страницу

## Технологии
- Python 3.10, Django 4.2
- PostgreSQL (запускается через docker-compose)
- Minio (хранение изображений и видео)
- Adminer (веб-интерфейс для управления БД)

## кратко ответы
1. Виды БД
•	Реляционные (SQL) – данные хранятся в таблицах со связями (PostgreSQL, MySQL).
•	Нереляционные (NoSQL) – документоориентированные (MongoDB), ключ-значение (Redis), графовые и т.д.
•	В лабораторной используется PostgreSQL (реляционная).
2. SQL запросы
•	DDL (создание структуры): CREATE TABLE, ALTER, DROP.
•	DML (манипуляция данными): SELECT, INSERT, UPDATE, DELETE.
•	В лабораторной: UPDATE bmstu_lab_bankrequest SET status = 'DELETED' ... (SQL).
3. Курсоры
•	Механизм для построчного чтения результатов SQL-запроса на сервере БД.
•	В Django: with connection.cursor() as cursor: – используется в delete_request для выполнения SQL без ORM.
4. ORM (Object-Relational Mapping)
•	Технология, позволяющая работать с БД через объекты языка программирования, не писать SQL.
•	В Django: модели BankService.objects.filter(...), get_or_create и т.д. – это ORM.
5. Модель и миграции
•	Модель – Python-класс, описывающий таблицу БД (поля, связи).
•	Миграции – файлы, которые Django создаёт для синхронизации моделей с БД (makemigrations, migrate). 
Позволяют изменять схему без потери данных.
6. Чистая архитектура
•	Принцип разделения на слои: представление (шаблоны), бизнес-логика (views), данные (модели). Каждый слой независим.
•	В Django это MVT (Model-View-Template) – вариант чистой архитектуры.
